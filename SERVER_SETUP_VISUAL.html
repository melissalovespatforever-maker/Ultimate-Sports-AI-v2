Server.js Setup Guide

Two lines of code to add database initialization

Overview

You need to add 2 things to your `/backend/server.js`:

  • ‚úì Import the database init routes (1 line)
  • ‚úì Register the routes in Express (1 line)
That's it! Then deploy and use the initializer HTML

1 Load the Database Init Routes

Location in server.js

Find this section around line 152-156:

/backend/server.js (line 152-156)
const tournamentsRoutes = safeRequire('./routes/tournaments', 'Tournaments'); const leaderboardsRoutes = safeRequire('./routes/leaderboards', 'Leaderboards');

Add this line:

const databaseInitRoutes = safeRequire('./routes/database-init', 'Database Init');
Add it RIGHT AFTER the leaderboards line - keep it with the other route imports

Result:

const tournamentsRoutes = safeRequire('./routes/tournaments', 'Tournaments'); const leaderboardsRoutes = safeRequire('./routes/leaderboards', 'Leaderboards'); const databaseInitRoutes = safeRequire('./routes/database-init', 'Database Init'); // ‚Üê NEW

2 Register the Routes

Location in server.js

Find this section around line 174-184:

/backend/server.js (line 174-184)
// Health Checks app.get('/health', (req, res) => res.status(200).json({ status: 'healthy', message: 'Backend running' })); app.get('/api/health', (req, res) => res.json({ status: 'healthy', service: 'ultimate-sports-ai-backend', version: '2.0.0', database: dbInitialized ? 'connected' : 'initializing' })); // ============================================ // PUBLIC ADMIN ENDPOINTS // ============================================

Add these lines:

// Database initialization routes (PUBLIC - no auth needed) app.use('/api/admin', databaseInitRoutes); // ‚Üê NEW
Add it BEFORE the comment about protected admin endpoints

Result:

// Health Checks app.get('/health', (req, res) => res.status(200).json({ status: 'healthy', message: 'Backend running' })); app.get('/api/health', (req, res) => res.json({ status: 'healthy', service: 'ultimate-sports-ai-backend', version: '2.0.0', database: dbInitialized ? 'connected' : 'initializing' })); // Database initialization routes (PUBLIC - no auth needed) app.use('/api/admin', databaseInitRoutes); // ‚Üê NEW // ============================================ // PUBLIC ADMIN ENDPOINTS // ============================================

3 Deploy to Railway

Push your changes

In your terminal, run:

git add . git commit -m "Add database initialization endpoints" git push
Railway will automatically rebuild and deploy your changes

Check deployment status:

  1. Go to https://railway.app
  2. Click your project
  3. Look for green checkmark (‚úì) next to your deployment
  4. Wait 1-2 minutes for deployment

4 Use the Initializer

Open the auto-initializer

Once deployed, open this in your browser:

init-database-auto.html

Then:

  1. Make sure your Railway backend URL is correct
  2. Click "Initialize Database"
  3. Watch the logs as it creates tables
  4. See the green ‚úÖ success message
‚úÖ Done! Your database is ready with all tables and ESPN data!

What Happens After

Once initialized, you get these new endpoints:

POST /api/admin/init-database

Creates all required tables

GET /api/scores/sync

Fetches ESPN data automatically

GET /api/scores/nfl

Returns NFL games from database

GET /api/scores/nba, /mlb, /nhl, /soccer

Get games for any sport
